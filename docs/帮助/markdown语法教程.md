📝 Markdown 语法指南
================

这份指南涵盖了 Markdown 的基础语法（Basic Syntax）和扩展语法（Extended Syntax），内容源自[Markdown Guide](https://www.markdownguide.org/)。

* * *

🛠️ 基础语法 (Basic Syntax)
-----------------------

几乎所有 Markdown 编辑器都支持以下基础元素。

### 1\. 标题 (Headings)

要创建标题，请在单词或短语前添加井号 (`#`)。`#` 的数量代表了标题的级别（对应 HTML 的 `<h1>` 到 `<h6>`）。

| Markdown 语法 | 渲染效果     |
| ----------- | -------- |
| `# 一级标题`    | **一级标题** |
| `## 二级标题`   | **二级标题** |
| `### 三级标题`  | **三级标题** |
| `#### 四级标题` | **四级标题** |

> **最佳实践：**
> 
> * 在 `#` 和标题文字之间**务必加一个空格**。
> * 建议在标题前后都留一个空行。

**替代语法：** 你也可以在文字下方添加 `===` (一级标题) 或 `---` (二级标题) 来创建标题。

    一级标题
    ===
    
    二级标题
    ---

### 2\. 段落与换行 (Paragraphs & Line Breaks)

* **段落：** 使用一个空行将文本分隔成不同的段落。
* **换行：** 在一行末尾添加**两个或更多空格**，然后按回车，可以强制换行（`<br>`）。

### 3\. 强调 (Emphasis)

你可以通过加粗、斜体或两者结合来强调文本。

| 效果        | 语法                  | 示例           |
| --------- | ------------------- | ------------ |
| **粗体**    | `**文字**` 或 `__文字__` | `**这是粗体**`   |
| _斜体_      | `*文字*` 或 `_文字_`     | `*这是斜体*`     |
| **_粗斜体_** | `***文字***`          | `***非常重要***` |
| 删除线       | `~~文字~~`            | `~~错误内容~~`   |

> **最佳实践：** 为了兼容性，建议在单词中间使用强调时（如 `Love**is**bold`），使用星号 `*` 而不是下划线 `_`。

### 4\. 引用 (Blockquotes)

使用 `>` 符号来创建引用块。

    > 这是一个引用块。
    >
    > 这是同一个引用块中的第二段。

**嵌套引用：**

    > 一级引用
    > > 二级嵌套引用

### 5\. 列表 (Lists)

#### 无序列表

使用短横线 `-`、星号 `*` 或加号 `+` 创建列表。

    - 第一项
    - 第二项
      - 嵌套项 (缩进 4 个空格或 1 个 Tab)
    - 第三项

#### 有序列表

使用数字加点 `1.` 创建。

    1. 第一项
    2. 第二项
    3. 第三项

> **注意：** 即使你所有的数字都写成 `1.`，渲染出来的结果也会自动排序为 1, 2, 3...

#### 任务列表 (Task Lists)

_属于扩展语法，但在 GitHub 等平台非常常用。_

    - [x] 已完成的任务
    - [ ] 待办任务

### 6\. 代码 (Code)

* **行内代码：** 使用反引号 `` ` `` 包裹。例如：`<div>`。
* **代码块：** 缩进 4 个空格或 1 个 Tab，或者使用围栏代码块（Fenced Code Blocks）。

**围栏代码块 (推荐)：** 使用三个反引号 \`\`\` 包裹代码，还可以指定语言以启用语法高亮。

    {
      "firstName": "John",
      "lastName": "Smith",
      "age": 25
    }

### 7\. 分割线 (Horizontal Rules)

使用三个或以上的 `*`、`-` 或 `_` 单独成行来创建分割线。

    ---

### 8\. 链接 (Links)

* **行内链接：** `[链接文字](URL "可选标题")`

* **参考式链接：** `[链接文字][引用ID]`，然后在文档其他地方定义 `[引用ID]: URL`
  
  访问 [Markdown Guide](https://www.markdownguide.org)。

### 9\. 图片 (Images)

语法与链接类似，但在最前面加一个感叹号 `!`。

    ![替代文本](图片路径 "可选标题")

### 10\. 转义字符 (Escaping Characters)

如果你想显示 Markdown 的保留字符（如 `*`, `#`, `[` 等），请在前面加反斜杠 `\`。

    \* 这不会被渲染为列表

* * *

🚀 扩展语法 (Extended Syntax)
-------------------------

并非所有编辑器都支持扩展语法，但现代编辑器（如 GitHub, Typora, Obsidian）通常都支持。

### 1\. 表格 (Tables)

使用管道 `|` 分隔列，使用短横线 `-` 分隔行和定义对齐方式。

    | 语法 | 描述 | 对齐方式 |
    | :--- | :---: | ---: |
    | Header | Title | 居左 |
    | Paragraph | Text | 居中 |
    | Footer | More | 居右 |

* `:---` 左对齐
* `:---:` 居中对齐
* `---:` 右对齐

### 2\. 脚注 (Footnotes)

允许你添加注释和引用，而不会弄乱文档正文。

    这是一个脚注示例[^1]。
    
    [^1]: 这是脚注的内容。

### 3\. 定义列表 (Definition Lists)

某些处理器支持术语和定义的列表。

    术语 1
    : 定义 1
    
    术语 2
    : 定义 2a
    : 定义 2b

### 4\. 高亮 (Highlight)

_注意：并非所有平台支持，部分需使用 HTML `<mark>` 标签。_

    ==高亮显示的文本==

### 5\. 下标与上标 (Subscript & Superscript)

* **下标：** 使用波浪号 `~`。例如：`H~2~O` → H2O
* **上标：** 使用插入符 `^`。例如：`X^2^` → X^2^

### 6\. 表情符号 (Emoji)

你可以直接复制粘贴 Emoji，或者使用简码（Shortcodes）。

    :heart: :smile: :tent:

_(注：简码支持取决于具体的应用程序)_

### 7\. 自动链接 (Automatic URL Linking)

大多数处理器会自动将 `http://` 或 `https://` 开头的 URL 转换为链接。

* * *

💻 HTML 支持
----------

许多 Markdown 编辑器允许直接使用 HTML 标签。如果 Markdown 语法无法满足需求（如调整颜色、复杂的表格），可以直接使用 HTML。

这是 红色的文字

    这是 <span style="color:red">红色的文字</span>。

> **注意：** 在块级 HTML 元素（如 `<div>`, `<table>`）中，通常不能使用 Markdown 语法。
